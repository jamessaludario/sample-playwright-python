Setup and teardown

The concepts

  • Setup — work done before a test runs: open a browser, log in, create a test user, load data.
  • Teardown — cleanup done after a test finishes (pass or fail): delete the test user, log out, close the browser.

Good teardown is what keeps tests independent: every test starts from a clean slate no matter what the previous test did.

pytest fixtures — how setup/teardown is written

In pytest, setup and teardown live in fixtures: functions marked with @pytest.fixture. A test "requests" a fixture simply by naming it as a parameter. Everything before the yield line is setup; everything after it is teardown.

conftest.py 
import pytest
from playwright.sync_api import Page


@pytest.fixture
def logged_in_page(page: Page):
    # ---------- SETUP (runs before the test) ----------
    page.goto("https://automationexercise.com/login")
    page.get_by_placeholder("Email Address").fill("tester@example.com")
    page.get_by_placeholder("Password").fill("secret123")
    page.get_by_role("button", name="Login").click()

    # yield hands control to the test, passing it the ready page
    yield page

    # ---------- TEARDOWN (runs after the test, even if it FAILED) ----------
    page.get_by_role("link", name="Logout").click()
tests/test_account.py 
from playwright.sync_api import expect


# Just naming "logged_in_page" as a parameter triggers the whole fixture.
def test_dashboard_greets_the_user(logged_in_page):
    expect(logged_in_page.get_by_text("Logged in as")).to_be_visible()
You've already been using a fixture! The page parameter in every earlier example is a fixture, provided by pytest-playwright. Its setup launches a fresh browser tab; its teardown closes it. That's why you never wrote "open browser" or "close browser" yourself.

Fixture scope — how often setup runs

ScopeSetup/teardown runs...Typical use
function (default)once per testfresh page, fresh test data
classonce per test classshared state for a group of related tests
moduleonce per fileexpensive setup shared by one file's tests
sessiononce per entire runstart a mock server, load config
@pytest.fixture(scope="session")
def api_token():
    # Fetched once, reused by every test in the run.
    return fetch_token_from_api()
Where fixtures live Put fixtures used by many files in conftest.py — pytest auto-discovers them, no import needed. Fixtures used by a single file can live at the top of that file.
