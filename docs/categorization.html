Categorization of tests

Why categorize?

Once you have dozens of tests, you rarely want to run all of them every time. After a small change you might run just the quick "smoke" checks (2 minutes) and save the full "regression" suite (30+ minutes) for later. Categories make that possible.

Markers — pytest's labels

A marker is a label you stick on a test with a decorator:

import pytest


@pytest.mark.smoke
def test_homepage_loads(page):
    ...

@pytest.mark.regression
def test_full_checkout_journey_with_new_address(page):
    ...

# A test can carry several markers:
@pytest.mark.smoke
@pytest.mark.cart
def test_add_item_to_cart(page):
    ...

Register your markers (avoid typo bugs)

Declare every marker in pytest.ini. Unregistered markers produce warnings, and — worse — a typo like @pytest.mark.smoek would otherwise silently create a new category and your test would stop running with the smoke suite.

pytest.ini (excerpt) 
[pytest]
markers =
    smoke: quick checks of the most critical user paths
    regression: the full, thorough suite
    cart: anything involving the shopping cart
addopts = --strict-markers   ; turn unknown markers into ERRORS - highly recommended

Running by category

# Only smoke tests
pytest -m smoke

# Everything EXCEPT slow regression tests
pytest -m "not regression"

# Smoke tests that also involve the cart
pytest -m "smoke and cart"

# Either category
pytest -m "smoke or cart"

Common suite names in QA teams

SuitePurposeWhen it runs
Smoke"Is the app fundamentally alive?" 5-15 fast tests of the most critical paths (login, homepage, checkout).On every code change / deployment.
Regression"Did anything anywhere break?" The full suite.Nightly, or before a release.
Feature areas (cart, auth, search...)Everything about one part of the app.When that area changes.

Other useful built-in markers

# Skip a test entirely (always give a reason!)
@pytest.mark.skip(reason="Feature removed in v2, test pending rewrite")
def test_old_wishlist(page): ...

# Skip only under some condition
@pytest.mark.skipif(sys.platform == "win32", reason="Safari not available on Windows")
def test_webkit_specific(page): ...

# "We KNOW this fails - there's an open bug." Reported as xfail, not failure.
@pytest.mark.xfail(reason="BUG-1234: totals rounding error")
def test_cart_total_rounding(page): ...
